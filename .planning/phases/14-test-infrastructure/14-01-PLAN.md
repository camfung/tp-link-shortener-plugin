1. Create `tests/e2e/scripts/cleanup_stress.py` as a standalone CLI script:
   - Shebang: `#!/usr/bin/env python3`
   - Docstring explaining both modes: `cleanup_stress.py <RUN_ID>` and `cleanup_stress.py --all-stress`
   - Uses argparse with mutually exclusive group: positional `run_id` OR `--all-stress` flag
   - `load_env()` function: reads `tests/e2e/.env` to get `TP_API_ENDPOINT` and `API_KEY` env vars (same .env loading pattern as conftest.py)
   - `delete_links_for_run(run_id, api_endpoint, api_key)`:
     - Reads `tests/e2e/data/stress_data_{run_id}.json`
     - For each link, tries DELETE `{api_endpoint}/items/{mid}` with `x-api-key` header first
     - If DELETE returns 4xx/5xx, falls back to PUT `{api_endpoint}/items/{mid}` with `{"status": "disabled"}`
     - Prints status for each MID (success/fail with status code)
     - Deletes the data JSON file after processing all links
   - `delete_all_stress(api_endpoint, api_key)`:
     - Globs `tests/e2e/data/stress_data_stress-*.json`
     - Calls `delete_links_for_run` for each found file (extracts run_id from filename)
   - `main()`: argparse dispatch to appropriate function
   - Uses `httpx.Client` (sync, not async — simple CLI script)
   - Print summary at end: N links processed, N files removed

Per user decision: cleanup is MANUAL ONLY — this script is never called from fixtures or pytest hooks. No automatic cleanup.
