<feature>
  <name>UsageMergeAdapter::merge()</name>
  <files>includes/TerrWallet/UsageMergeAdapter.php, tests/Unit/TerrWallet/UsageMergeAdapterTest.php</files>
  <behavior>
    Full outer join of usage days and wallet transactions by date key.

    Input:
    - $usageDays: array of associative arrays with keys {date, totalHits, hitCost, balance}
    - $walletTransactions: array of WalletTransaction objects (->date already YYYY-MM-DD from DTO)

    Output: indexed array of merged day records, sorted ascending by date. Each record:
    {date, totalHits, hitCost, balance, otherServices}

    otherServices shape (per CONTEXT.md locked decisions):
    - When wallet transactions exist for that date: {amount: float (rounded to 2dp), items: [{amount: float, description: string}, ...]}
    - When no wallet transactions for that date: null

    Cases:
    - merge([day1], [tx_day1]) -> [{...day1, otherServices: {amount, items}}] (MERGE-01)
    - merge([day1], [tx_day1_a, tx_day1_b]) -> [{...day1, otherServices: {amount: sum, items: [a, b]}}] (MERGE-02)
    - merge([], [tx_day3]) -> [{date: day3, totalHits: 0, hitCost: 0.00, balance: 0.00, otherServices: {...}}] (MERGE-03)
    - merge([day1, day2], []) -> [{...day1, otherServices: null}, {...day2, otherServices: null}] (empty wallet)
    - merge([], []) -> [] (both empty)
    - merge([day2, day1], [tx_day3]) -> sorted [day1, day2, day3] regardless of input order
    - Items with empty/whitespace descriptions: store "" in description field (preserve item for amount tracking)
    - Float precision: round(otherServices.amount, 2) after summing to avoid IEEE 754 drift
  </behavior>
  <implementation>
    Create stateless class UsageMergeAdapter in TerrWallet namespace with one public static method:

    ```php
    public static function merge(array $usageDays, array $walletTransactions): array
    ```

    Algorithm (hash map full outer join):
    1. Seed associative array keyed by date from $usageDays, each with otherServices: null
    2. Iterate $walletTransactions, grouping by $tx->date:
       - If date not in map, create zero-filled usage record
       - Initialize otherServices if null for that date
       - Add to items array, accumulate amount
    3. After all transactions processed, round each otherServices.amount to 2 decimal places
    4. Extract values, usort ascending by date, return indexed array

    Do NOT mutate input arrays. Do NOT filter out items with empty descriptions (store "" per CONTEXT.md decision). Use strcmp() for date sorting (lexicographic on YYYY-MM-DD is correct).

    The class lives at includes/TerrWallet/UsageMergeAdapter.php and is autoloaded via the existing PSR-4 TerrWallet autoloader registered in includes/autoload.php.

    Test file at tests/Unit/TerrWallet/UsageMergeAdapterTest.php following existing PHPUnit conventions (namespace Tests\Unit\TerrWallet, extends TestCase).
  </implementation>
</feature>
