   - Module docstring with TP-71 bug description from Jira
   - Class `TestTP71` with `@pytest.mark.regression_bugs`

   - Reuse the `create_single_link` pattern from stress tests (copy the helper function into the test file or import from a shared location)
   - `test_destination_matches_submission(self, client_links_page)`:
     a. Generate unique keyword: `reg-tp71-{uuid.uuid4().hex[:6]}`
     b. Create a link via UI with destination "https://example.com/original"
     c. Verify the created link's destination shows "https://example.com/original" (inspect the table row or edit modal)
     d. Assert the stored destination matches what was submitted (not a cached/stale value)
   - `test_destination_updates_not_cached(self, client_links_page)`:
     a. Create a link with destination "https://example.com/before"
     b. Open edit modal for that link
     c. Change destination to "https://example.com/after"
     d. Save the edit
     e. Reopen the edit modal and verify destination is "https://example.com/after" (not the old cached value)
   - Use `page.wait_for_selector()` and `page.wait_for_function()` instead of `time.sleep()` for all waits
