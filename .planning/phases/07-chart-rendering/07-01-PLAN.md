**CSS changes** in `assets/css/usage-dashboard.css`:

Update the `.tp-ud-chart-wrapper` rule (currently at ~line 187) to add `min-width: 0` and `height: 280px`:

```css
.tp-ud-chart-wrapper {
    position: relative;   /* Required by Chart.js for responsive sizing */
    width: 100%;
    min-width: 0;         /* Prevents flex child infinite resize loop (Chart.js #5805, #9001) */
    height: 280px;        /* Explicit height -- NOT min-height -- gives Chart.js a definitive constraint */
    margin-bottom: 24px;
}
```

2. **Add `renderChart(data)` function** after `renderSummaryCards()` and before `showEmptyState()`. The complete function:
   - Guard: return early if `document.getElementById('tp-ud-chart')` is null
   - Guard: return early if `typeof Chart === 'undefined'` (CDN load failure protection)
   - Destroy previous instance: `if (state.chart) { state.chart.destroy(); state.chart = null; }`
   - On empty data (`!data || data.length === 0`): return after destroy (leave canvas blank)
   - Build three arrays by iterating `data`: `labels[]` from `data[i].date`, `clicksData[]` and `qrData[]` from `splitHits(data[i].totalHits)`
   - Create chart: `state.chart = new Chart(ctx, config)` with:
     - `type: 'line'`
     - Two datasets:
       - **Clicks (est.)**: `borderColor: '#f5a623'`, `backgroundColor: 'rgba(245, 166, 35, 0.15)'`, `fill: 'origin'`, `tension: 0.3`, `pointRadius: 4`, `pointBackgroundColor: '#f5a623'`, `pointBorderColor: '#f5a623'`, `pointHoverRadius: 6`, `borderWidth: 2`, `order: 2`
       - **QR Scans (est.)**: `borderColor: '#22b573'`, `backgroundColor: 'rgba(34, 181, 115, 0.12)'`, `fill: 'origin'`, `tension: 0.3`, `pointRadius: 4`, `pointBackgroundColor: '#22b573'`, `pointBorderColor: '#22b573'`, `pointHoverRadius: 6`, `borderWidth: 2`, `order: 1`
     - Options:
       - `responsive: true`, `maintainAspectRatio: false`
       - `interaction: { mode: 'index', intersect: false }`
       - Legend: `position: 'top'`, font family `'Poppins', sans-serif`, size 12, `usePointStyle: true`, padding 16
       - Tooltip: `backgroundColor: 'rgba(30, 47, 80, 0.9)'`, Poppins font, padding 10, cornerRadius 6
       - Y-axis: `stacked: true`, `beginAtZero: true`, `ticks.precision: 0`, grid color `'rgba(207, 226, 255, 0.4)'`
       - X-axis: `ticks.maxTicksLimit: 15`, `maxRotation: 45`, `minRotation: 0`, `grid.display: false`

   Use the exact code from the research document's "Complete renderChart() Function" section as the reference implementation.

3. **Integrate renderChart into loadData() success callback** (~line 489):
   - In the `state.data.length === 0` branch, add `renderChart([]);` to clear the chart
   - In the else branch, add `renderChart(state.data);` after `renderSummaryCards(state.data)` and before `renderTable()`

**Important notes:**
- Do NOT call `renderChart()` from sort handlers or pagination -- chart shows full date range, not paginated data
- The `(est.)` suffix in dataset labels satisfies CHART-05 (estimated disclaimer in chart legend)
- Use `fill: 'origin'` (not `fill: true` or `fill: 'stack'`) -- origin fills from line to X-axis zero
- The `order` property controls draw order: clicks (order 2) drawn first (behind), QR (order 1) drawn second (in front)
