  <action>
Create `tests/e2e/stress/test_verify_dashboard.py` with a test marked `@pytest.mark.stress`.

**Constants:**
- `POLL_TIMEOUT = int(os.getenv("STRESS_POLL_TIMEOUT", "120"))` (seconds)
- `POLL_INTERVAL = int(os.getenv("STRESS_POLL_INTERVAL", "10"))` (seconds)
- Import `BASE_URL`, `USAGE_DASHBOARD_PATH` from env vars (same pattern as conftest.py)
- Import `datetime` for today's date

**Helper function `poll_for_usage_data(page, timeout, interval)`:**
- Get today's date as ISO string: `datetime.date.today().isoformat()`
- Set start date and end date inputs to today (covers stress test activity):
  - `page.fill("#tp-ud-start-date", today_str)`
  - `page.fill("#tp-ud-end-date", today_str)`
- Click the Apply button: `page.click("#tp-ud-apply-btn")`
- Poll loop: `elapsed = 0` while `elapsed < timeout`:
  - Wait for content to load (not skeleton): wait for `#tp-ud-content` to be visible or `#tp-ud-tbody` to have children
  - Check table body `#tp-ud-tbody` for rows: `rows = page.query_selector_all("#tp-ud-tbody tr")`
  - If rows exist, check if any row has non-zero hit count:
    - For each row, get the totalHits cell text (column index based on table structure)
    - If any row has hits > 0, return True with row count
  - If no data yet, wait `interval` seconds
  - Re-apply date filter to trigger a fresh AJAX fetch (important -- the cached response may be stale):
    - `page.click("#tp-ud-apply-btn")`
  - Increment elapsed by interval
- If timeout reached, return False with diagnostic info

**Main test function `test_verify_dashboard(auth_context)`:**
- Create a page from `auth_context`
- Navigate to `{BASE_URL}{USAGE_DASHBOARD_PATH}`
- Wait for `.tp-ud-container` to be visible (timeout 15s)

- **Test 1: Page loads with dashboard structure (VERIFY-01)**
  - Assert `.tp-ud-container` is visible
  - Assert `#tp-ud-start-date` input exists
  - Assert `#tp-ud-end-date` input exists

- **Test 2: Usage table shows records (VERIFY-02)**
  - Call `poll_for_usage_data(page, POLL_TIMEOUT, POLL_INTERVAL)`
  - Assert the poll returned True (data found)
  - Assert row count > 0

- **Test 3: Chart renders with data (VERIFY-03)**
  - Assert canvas element exists: `page.query_selector("#tp-ud-chart")` or `page.query_selector("canvas")` is not None
  - Optionally check chart has non-zero dimensions (width > 0, height > 0)

- **Test 4: Polling was used, not hardcoded sleep (VERIFY-04)**
  - This is structural (the poll_for_usage_data function handles it)
  - The test itself should log how many poll iterations were needed

- Close the page

**Key design decisions (from research Pitfall 4):**
- Re-apply date filter on each poll iteration to trigger fresh AJAX fetch (avoid stale cached data)
- Generous timeout (120s default) with 10s intervals for eventual consistency
- Log poll progress for debugging failed runs

**Do NOT:**
- Use `time.sleep(60)` or any hardcoded wait before checking data
- Assume data is immediately available after usage generation
- Check for specific link keywords in the table (the dashboard shows aggregate daily data, not per-link)
