  <action>
Create `run_stress.sh` at the project root. Make it executable (`chmod +x`).

**Script structure:**

```
#!/usr/bin/env bash
set -euo pipefail
```

1. **Generate or accept RUN_ID:**
   - If `$1` is provided, use it as RUN_ID
   - Otherwise generate: `RUN_ID="stress-$(openssl rand -hex 4)"`
   - Export `STRESS_RUN_ID=$RUN_ID` so pytest fixtures pick it up

2. **Print banner:**
   - Show RUN_ID, timestamp, link count (from `STRESS_LINK_COUNT` env var, default 50), hits per link (from `STRESS_HITS_PER_LINK`, default 5)

3. **Stage 1: Link Creation**
   - Print stage header: "Stage 1/3: Creating links..."
   - Run: `cd tests/e2e && python -m pytest stress/test_create_links.py -m stress -v -s --timeout=600`
   - The `--timeout=600` gives 10 minutes for 50 link creations
   - If exit code != 0: print "FAILED: Link creation" and exit 1

4. **Stage 2: Usage Generation**
   - Print stage header: "Stage 2/3: Generating usage traffic..."
   - Run: `cd tests/e2e && python -m pytest stress/test_generate_usage.py -m stress -v -s --timeout=900`
   - The `--timeout=900` gives 15 minutes for usage generation with rate limiting
   - If exit code != 0: print "FAILED: Usage generation" and exit 1

5. **Stage 3: Dashboard Verification**
   - Print stage header: "Stage 3/3: Verifying dashboard..."
   - Run: `cd tests/e2e && python -m pytest stress/test_verify_dashboard.py -m stress -v -s --timeout=300`
   - The `--timeout=300` gives 5 minutes for polling verification
   - If exit code != 0: print "FAILED: Dashboard verification" and exit 1

6. **Success summary:**
   - Print "ALL STAGES PASSED" with RUN_ID and timestamp

7. **Cleanup prompt:**
   - Ask: "Run cleanup? (y/N)"
   - If yes: `python tests/e2e/scripts/cleanup_stress.py "$RUN_ID"`
   - If no: print "Skipping cleanup. Run manually: python tests/e2e/scripts/cleanup_stress.py $RUN_ID"

**Environment variables documented in script header comment:**
- `STRESS_RUN_ID` - Override run ID (default: random)
- `STRESS_LINK_COUNT` - Number of links to create (default: 50)
- `STRESS_HITS_PER_LINK` - Hits per link (default: 5)
- `STRESS_RATE_LIMIT` - Delay between requests in seconds (default: 1.5)
- `STRESS_POLL_TIMEOUT` - Dashboard verification timeout in seconds (default: 120)
- `TP_SHORT_DOMAIN` - Short link domain (default: dev.trfc.link)
- `TP_API_ENDPOINT` - API endpoint for cleanup (required for cleanup)
- `API_KEY` - API key for cleanup (required for cleanup)

**Do NOT:**
- Run stages in parallel (they are sequential dependencies)
- Skip the cleanup prompt (user should decide)
- Hardcode any environment-specific values
