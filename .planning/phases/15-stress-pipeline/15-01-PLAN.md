  <action>
Create `tests/e2e/stress/test_create_links.py` with a single test function marked `@pytest.mark.stress`.

**Test structure:**

1. **Setup:** Import `json`, `os`, `pytest`, and Playwright types. Define constants:
   - `LINK_COUNT = int(os.getenv("STRESS_LINK_COUNT", "50"))`
   - `DESTINATION_URL = "https://example.com"`
   - Use `BASE_URL` and `CLIENT_LINKS_PATH` from `tests/e2e/conftest.py` (import or re-read from env)

2. **Helper function `create_single_link(page, keyword, destination_url)`:**
   - Click `#tp-cl-add-link-btn` to open the Add Link modal
   - Wait for modal overlay `#tp-cl-edit-modal-overlay` to be visible (timeout 10s)
   - Fill `#tp-destination` with `destination_url`
   - Wait for async URL validation to complete: wait for `#tp-custom-key-group` to be visible (timeout 15s -- accounts for debounced validation AJAX call)
   - Clear and fill `#tp-custom-key` with `keyword`
   - Set up response interception BEFORE clicking submit: use `with page.expect_response(lambda r: "admin-ajax.php" in r.url and r.request.post_data and "tp_create_link" in r.request.post_data)` context manager
   - Inside the context manager, click `#tp-submit-btn`
   - Parse the response JSON to extract `data.mid` (the link's MID)
   - Wait for modal to close or success indicator (wait for `#tp-cl-edit-modal-overlay` to be hidden, timeout 10s)
   - Return dict `{"keyword": keyword, "url": destination_url, "mid": mid}`

3. **Main test function `test_create_stress_links(auth_context, run_id, stress_data_file)`:**
   - Create a page from `auth_context`
   - Navigate to `{BASE_URL}{CLIENT_LINKS_PATH}`
   - Wait for `.tp-cl-container` to load (timeout 15s)
   - Loop `LINK_COUNT` times (i from 0 to LINK_COUNT-1):
     - Generate keyword: `f"{run_id}-{i:03d}"` (e.g., `stress-a1b2c3d4-000`)
     - Call `create_single_link(page, keyword, DESTINATION_URL)`
     - Append result to `created_links` list
     - Print progress: `f"Created {i+1}/{LINK_COUNT}: {keyword}"`
   - After loop, write `created_links` to `stress_data_file` as JSON (indent=2)
   - Assert `len(created_links) == LINK_COUNT`
   - Close the page

**Critical pitfall handling (from research):**
- **Pitfall 3 (Multiple AJAX calls):** The `expect_response` predicate filters by `tp_create_link` in `post_data` to avoid capturing URL validation or key validation AJAX calls.
- **Pitfall 6 (Form state not reset):** After each link creation, the modal closes. Re-clicking `#tp-cl-add-link-btn` triggers `tp:resetForm` which calls `switchToCreateMode()`. Add a small wait (500ms) between creations to let animations complete.
- **Pitfall 1 (Custom key field not visible):** The 15s timeout on `#tp-custom-key-group` visibility accounts for the debounced validation delay.
- **Pitfall 2 (Response interception timing):** The `with page.expect_response()` context manager is entered BEFORE clicking submit.

**Do NOT:**
- Use shortcode auto-generation (known broken with 500 errors)
- Attempt parallel browser contexts for creation (UI flow is inherently sequential)
- Use hardcoded sleeps longer than 500ms between creations
