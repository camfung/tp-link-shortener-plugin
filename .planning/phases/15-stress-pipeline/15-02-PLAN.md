  <action>
Create `tests/e2e/stress/test_generate_usage.py` with an async test function marked `@pytest.mark.stress`.

**Constants and configuration:**
- `SHORT_DOMAIN = os.getenv("TP_SHORT_DOMAIN", "dev.trfc.link")`
- `HITS_PER_LINK = int(os.getenv("STRESS_HITS_PER_LINK", "5"))`
- `MAX_CONCURRENCY = int(os.getenv("STRESS_MAX_CONCURRENCY", "5"))`
- `MAX_RETRIES = 3`
- `USER_AGENT = "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36"`

**Helper async function `hit_link(client, url, semaphore, rate_delay, stats)`:**
- Acquire semaphore before making request
- Send GET to `url` with `follow_redirects=False` (just need the 301/302 to register usage -- don't load destination)
- Set `User-Agent` header to `USER_AGENT` (avoid bot filtering)
- On success (any 2xx or 3xx status): increment `stats["success"]`
- On 429: exponential backoff (2^attempt seconds, up to MAX_RETRIES), increment `stats["retried"]`
- On other error: increment `stats["failed"]`, log the error
- After each request, `await asyncio.sleep(rate_delay)` for rate limiting

**Main test function `test_generate_usage(stress_links, stress_rate_limit)`:**
- Mark with `@pytest.mark.stress` and `@pytest.mark.asyncio`
- Read links from `stress_links` fixture (auto-skips if data file missing)
- Build URL list: `[f"https://{SHORT_DOMAIN}/{link['keyword']}" for link in stress_links]`
- Create `asyncio.Semaphore(MAX_CONCURRENCY)` for concurrency control
- Initialize stats dict: `{"success": 0, "failed": 0, "retried": 0, "total": len(stress_links) * HITS_PER_LINK}`
- Create `httpx.AsyncClient(timeout=30.0, verify=False)` (verify=False for dev cert)
- For each hit round (0 to HITS_PER_LINK-1):
  - Create tasks for all links in this round using `asyncio.gather()`
  - Print progress: `f"Round {round+1}/{HITS_PER_LINK}: {len(stress_links)} requests"`
- After all rounds complete, print summary stats
- Assert `stats["success"] > 0`, with message showing stats
- Assert `stats["failed"] == 0` or warn if some failed (non-fatal -- usage may still register)

**Key design decisions:**
- `follow_redirects=False`: The 301/302 redirect response alone registers usage. Loading the destination page is unnecessary overhead.
- `asyncio.Semaphore`: Limits concurrent connections to avoid overwhelming the redirect service.
- Per-request delay via `stress_rate_limit` fixture (default 1.5s): Conservative to avoid hitting rate limits.
- Realistic `User-Agent`: Some redirect services filter requests without a browser UA.

**Do NOT:**
- Use `follow_redirects=True` (wastes bandwidth loading destination pages)
- Use pytest-xdist for this test (the async concurrency within the test is sufficient)
- Skip rate limiting (will cause 429s on the redirect service)
