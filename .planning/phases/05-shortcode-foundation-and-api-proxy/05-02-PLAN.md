    **1. Add `getUserActivitySummary()` to `TrafficPortalApiClient.php`**

    Follow the `getUserMapItems()` method pattern exactly. Add the method after the existing public methods:

    ```php
    public function getUserActivitySummary(int $uid, string $startDate, string $endDate): array
    ```

    Implementation:
    - Log request start: `=== GET USER ACTIVITY SUMMARY REQUEST ===`
    - Build query params: `['start_date' => $startDate, 'end_date' => $endDate]`
    - Build URL: `$this->apiEndpoint . '/user-activity-summary/' . $uid . '?' . http_build_query($queryParams)`
    - Use `$this->getHttpClient()` to make a GET request with headers: `Content-Type: application/json` and `x-api-key: $this->apiKey` (matching the existing header pattern in other methods)
    - Get status code and body from response
    - If body is empty, throw `new NetworkException('Empty response from API')`
    - JSON decode; if decode fails, throw `new ApiException('Invalid JSON response: ' . json_last_error_msg(), $httpCode)`
    - Call `$this->handleHttpErrors($httpCode, $data)` (existing method handles 401, 400, 429, 500, 502)
    - Return the decoded array
    - Use timeout of 15 seconds: pass `['timeout' => 15]` in the request options (Claude's discretion per research)

    **2. Register AJAX handlers in `class-tp-api-handler.php`**

    **3. Add `ajax_get_usage_summary()` method to `class-tp-api-handler.php`**

    Add after the existing `ajax_get_link_history()` method. Follow the existing AJAX handler pattern:

    ```php
    public function ajax_get_usage_summary(): void
    ```

    Implementation:
    a) Verify nonce: `check_ajax_referer('tp_link_shortener_nonce', 'nonce');`
    b) Check login: `if (!is_user_logged_in())` -> `wp_send_json_error(['message' => __('You must be logged in.', 'tp-link-shortener')], 401); return;`
    c) Get UID server-side: `$uid = TP_Link_Shortener::get_user_id();` (DATA-02: NEVER from $_POST)
    d) Sanitize date inputs from $_POST: `sanitize_text_field()` for `start_date` and `end_date`. Validate format with `preg_match('/^\d{4}-\d{2}-\d{2}$/', ...)`. Return 400 error if invalid format.
    e) Try/catch: call `$this->client->getUserActivitySummary($uid, $start_date, $end_date)`
    f) On success: validate and reshape the response with `$this->validate_usage_summary_response($raw)`, then `wp_send_json_success($validated)`
    g) Catch `NetworkException` -> `$this->send_usage_proxy_error($e, 'network')`
    h) Catch `ApiException` -> `$this->send_usage_proxy_error($e, 'api')`
    i) Catch `\Exception` -> `$this->send_usage_proxy_error($e, 'unknown')`

    **4. Add `validate_usage_summary_response()` private method**

    LOCKED DECISION: Proxy validates and reshapes the API response. The external API returns `{ message, success, source: [...] }` where `source` contains the data array.

    ```php
    private function validate_usage_summary_response(array $raw): array
    ```

    - Extract `$source = $raw['source'] ?? [];`
    - If `$source` is not an array, set to empty array
    - Loop through `$source`, for each record:
      - Skip if `date` or `totalHits` is missing
      - Build validated record: `date` -> `sanitize_text_field()`, `totalHits` -> `(int)`, `hitCost` -> `(float)` (default 0), `balance` -> `(float)` (default 0)
    - Return `['days' => $days]` (reshape: unwrap `source` key, rename to `days` for frontend clarity)

    **5. Add `send_usage_proxy_error()` private method**

    LOCKED DECISION: Generic error for regular users, detailed error for admins.

    private function send_usage_proxy_error(\Exception $e, string $type): void

    - Build response: `['message' => __('Unable to load usage data. Please try again.', 'tp-link-shortener')]`

    - Log the error: `error_log('TP Link Shortener: Usage summary error (' . $type . '): ' . $e->getMessage());`
    - `wp_send_json_error($response);`

  <action>
    Create `assets/js/usage-dashboard.js` following the `client-links.js` jQuery IIFE pattern.

    **Structure:**
    ```javascript
    (function($) {
        'use strict';

        // State object
        // DOM cache
        // Init function
        // Data loading function
        // State toggle functions (showSkeleton, hideSkeleton, showError, hideError, showContent, hideContent)
        // Event binding
        // Document ready
    })(jQuery);
    ```

    **State object:**
    ```javascript
    var state = {
        isLoading: false,
        dateStart: tpUsageDashboard.dateRange.start,
        dateEnd: tpUsageDashboard.dateRange.end,
        data: null
    };
    ```

    **DOM cache** (in `init` or `cacheElements` function):
    Cache jQuery references to: `#tp-ud-skeleton`, `#tp-ud-error`, `#tp-ud-error-msg`, `#tp-ud-content`, `#tp-ud-retry`, `#tp-ud-admin-error` (for admin detail, if it exists).

    **`loadData()` function:**
    - Guard: `if (state.isLoading) return;`
    - Set `state.isLoading = true;`
    - Call `showSkeleton()`, `hideError()`, `hideContent()`
    - jQuery `$.ajax()` POST to `tpUsageDashboard.ajaxUrl` with data:
      - `action: 'tp_get_usage_summary'`
      - `nonce: tpUsageDashboard.nonce`
      - `start_date: state.dateStart`
      - `end_date: state.dateEnd`
      - NOTE: NO uid field -- server determines it (DATA-02)
    - `timeout: 20000` (20s, allowing 15s server timeout + network overhead)
    - Success callback:
      - `state.isLoading = false;`
      - If `response.success && response.data && response.data.days`:
        - `state.data = response.data.days;`
        - `hideSkeleton();`
        - If `state.data.length === 0`: show a "no data" message inside content area, then `showContent()`
        - Else: `showContent()` (actual table/chart rendering is Phases 6-7; for now just show the content div)
      - Else: extract error message and call `showError(msg)`
    - Error callback:
      - `state.isLoading = false;`
      - `hideSkeleton();`
      - `showError(tpUsageDashboard.strings.error);`

    **State toggle functions:**
    - `showSkeleton()`: show `#tp-ud-skeleton`, hide `#tp-ud-error` and `#tp-ud-content`
    - `hideSkeleton()`: hide `#tp-ud-skeleton`
    - `showError(msg)`: set `#tp-ud-error-msg` text, show `#tp-ud-error`
      - If `tpUsageDashboard.isAdmin` and response has `error_type`/`error_detail`: append admin detail in a small muted paragraph below the main error message
    - `hideError()`: hide `#tp-ud-error`
    - `showContent()`: show `#tp-ud-content`
    - `hideContent()`: hide `#tp-ud-content`

    **Event binding:**
    - `#tp-ud-retry` click: call `loadData()` (LOCKED DECISION: re-fetch without page reload)

    **Document ready:**
    ```javascript
    $(document).ready(function() {
        cacheElements();
        bindEvents();
        loadData();  // Auto-fetch on page load
    });
    ```

    **Error detail for admins:**
    When `showError` is called from the success callback with `response.success === false`, check if `response.data.error_type` exists AND `tpUsageDashboard.isAdmin` is truthy. If so, append a `<p class="tp-ud-error-detail text-muted small">` with the error type and detail below the main message. When `showError` is called from the error callback (network failure), just show the generic message.
