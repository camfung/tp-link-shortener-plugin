  <action>
    **1. Create `includes/class-tp-usage-dashboard-shortcode.php`**

    Follow the exact pattern from `class-tp-client-links-shortcode.php`. The class must:

    - Guard with `if (!defined('ABSPATH')) { exit; }` and `declare(strict_types=1);`
    - Constructor: `add_shortcode('tp_usage_dashboard', [$this, 'render_shortcode']);`
    - `render_shortcode($atts): string` method:
      - **Auth gate (LOCKED DECISION):** If `!is_user_logged_in()`, return `wp_login_form()` wrapped in a `<div class="tp-ud-login-wrapper">`. CRITICAL: pass `'echo' => false` so it returns a string (shortcodes must return, not echo). Pass `'redirect' => get_permalink()` so user returns to this page after login. Pass `'remember' => true`.
      - For logged-in users: `shortcode_atts(['days' => 30], $atts)`, call `$this->enqueue_assets($atts)`, then `ob_start()` / `include template` / `return ob_get_clean()`.
    - `enqueue_assets(array $atts): void` method following the client-links pattern:
      - Enqueue: `tp-bootstrap` CSS (5.3.0 CDN), `tp-fontawesome` CSS (6.4.0 CDN), `tp-link-shortener` base CSS, `tp-usage-dashboard` CSS (depends on bootstrap + base).
      - Enqueue: `tp-bootstrap-js` (depends on jquery), `tp-chartjs` (4.4.1 CDN -- needed for canvas placeholder; actual chart is Phase 7), `tp-usage-dashboard-js` (depends on jquery, bootstrap-js, chartjs). All scripts in footer (`true`).
      - `wp_localize_script('tp-usage-dashboard-js', 'tpUsageDashboard', [...])` AFTER enqueuing the script. Include: `ajaxUrl`, `nonce` (using `tp_link_shortener_nonce`), `isLoggedIn`, `isAdmin` (via `current_user_can('manage_options')`), `dateRange` (computed from `$atts['days']`), and `strings` (loading, error, noData, retry).

    **2. Create `templates/usage-dashboard-template.php`**

    Three-state template (all wrapped in `<div class="tp-ud-container">`):

    a) **Loading skeleton** (`#tp-ud-skeleton`, visible by default): Use CSS pulse animation placeholders. Include a chart placeholder area (a rounded rectangle about 300px tall), a summary stats strip placeholder (3-4 small rectangles inline), and table row placeholders (5 rows of horizontal bars). Use `tp-ud-skeleton-*` class names.

    b) **Error state** (`#tp-ud-error`, `style="display: none;"`): A centered message container with a `<p id="tp-ud-error-msg">` for the error text, and a Bootstrap-styled "Retry" button (`#tp-ud-retry`) with a Font Awesome `fa-redo` icon. Admins will see additional error detail injected by JS (Phase 5 Plan 02).

    c) **Content state** (`#tp-ud-content`, `style="display: none;"`): Placeholder containers that Phases 6-8 will fill:
      - Date range header area with two date inputs (type="date") and an "Apply" button (wired in Phase 8, but rendered now as part of the skeleton layout).
      - Summary stats strip placeholder div (`#tp-ud-summary-strip`).
      - Chart canvas: `<canvas id="tp-ud-chart"></canvas>` inside a responsive wrapper div.
      - Stats table placeholder div (`#tp-ud-table-container`).

    **3. Create `assets/css/usage-dashboard.css`**

    Scoped styles using `tp-ud-` prefix to avoid conflicts:
    - `.tp-ud-container`: max-width 1200px, centered, padding.
    - `.tp-ud-login-wrapper`: centered, max-width 400px, padding for the login form.
    - `.tp-ud-skeleton-*`: Gray background (`#e0e0e0`), border-radius, CSS `@keyframes pulse` animation (opacity oscillation from 0.6 to 1.0, 1.5s infinite).
    - `.tp-ud-skeleton-chart`: height ~250px, margin-bottom.
    - `.tp-ud-skeleton-row`: height ~20px, margin-bottom ~12px, varying widths (100%, 85%, 70%, 95%, 60%) for visual variety.
    - `.tp-ud-skeleton-stats`: display flex, gap, three placeholder blocks ~120px wide x 60px tall.
    - `.tp-ud-error`: text-align center, padding, muted color.
    - `.tp-ud-content`: standard spacing for content sections.
    - Chart wrapper: position relative, width 100%.

    **4. Register in plugin entry and main class**

    In `tp-link-shortener.php`: Add `require_once TP_LINK_SHORTENER_PLUGIN_DIR . 'includes/class-tp-usage-dashboard-shortcode.php';` after the `class-tp-client-links-shortcode.php` require.

    In `includes/class-tp-link-shortener.php`:
    - Add a private property `$usage_dashboard_shortcode;` after the `$client_links_shortcode` property.
    - In the `init()` method, add `$this->usage_dashboard_shortcode = new TP_Usage_Dashboard_Shortcode();` after the client links shortcode instantiation.

    **IMPORTANT:** Do NOT create the JS file (`assets/js/usage-dashboard.js`) in this task -- it is created in Plan 02. The `wp_enqueue_script` call for it will produce a "file not found" warning in dev until Plan 02 runs, which is acceptable. Alternatively, create a minimal stub JS file containing only the jQuery IIFE wrapper: `(function($) { 'use strict'; })(jQuery);`

    **5. Create unit tests: `tests/Unit/TrafficPortal/UsageDashboardShortcodeTest.php`**

    Follow the existing PHPUnit test patterns in `tests/Unit/TrafficPortal/`. Since WordPress functions (`add_shortcode`, `wp_login_form`, `is_user_logged_in`, etc.) are not available in pure unit tests, focus on testing the class structure and any pure logic:

    - Test that the class file can be required without errors (syntax check)
    - Test that the `render_shortcode` method exists on the class
    - Test that `enqueue_assets` method exists on the class
    - Use reflection to verify the constructor registers the shortcode hook

    **Note:** Full integration tests (actually calling `render_shortcode`) require WordPress bootstrapping and are covered by the E2E tests in Plan 05-03. Keep unit tests focused on what can be tested without WordPress.
